# Snowflake dbt Projects

Complete reference for managing dbt projects natively on Snowflake, including SQL commands, Snowflake CLI (`snow dbt`), project configuration, deployment workflows, and CI/CD integration.

## When to Activate

- Creating or managing dbt projects on Snowflake
- Deploying dbt models to Snowflake from local machine or CI/CD
- Writing SQL commands for dbt project lifecycle (CREATE, ALTER, EXECUTE, DROP)
- Using Snowflake CLI (`snow dbt`) for local development workflows
- Configuring `profiles.yml` or `dbt_project.yml` for Snowflake
- Scheduling dbt runs with Snowflake Tasks
- Setting up Git integrations for dbt on Snowflake
- Managing dbt project versions and deployments

---

## Architecture Overview

Snowflake natively supports dbt projects as first-class objects. There are two primary interaction paths:

1. **SQL Commands** — Manage dbt project objects directly in Snowflake (CREATE/ALTER/EXECUTE/DROP DBT PROJECT)
2. **Snowflake CLI** (`snow dbt`) — Deploy and execute dbt projects from your local machine or CI/CD pipelines

Both paths create **dbt project objects** in Snowflake that store your dbt code, can be versioned, and can be executed on-demand or on a schedule.

## Project Structure

A valid Snowflake dbt project requires at minimum:

```
my_dbt_project/
├── dbt_project.yml          # Project configuration (profile reference, vars, etc.)
├── profiles.yml             # Connection profiles (dev/prod targets)
├── packages.yml             # Optional: dbt package dependencies
├── package_lock.yml         # Auto-generated by `dbt deps`
├── models/
│   ├── staging/
│   │   └── stg_orders.sql
│   ├── marts/
│   │   └── customer_metrics.sql
│   └── schema.yml           # Model tests and documentation
├── macros/
├── seeds/
├── snapshots/
└── tests/
```

## Configuration Files

### profiles.yml (Snowflake-native dbt)

When running dbt natively on Snowflake, `account` and `user` fields are not needed (Snowflake injects them). Passwords are **not supported** and will cause deployment failures.

```yaml
my_project:
  target: dev
  outputs:
    dev:
      type: snowflake
      account: 'not needed'      # Ignored when running natively
      user: 'not needed'         # Ignored when running natively
      role: my_role
      database: my_database
      schema: dev
      warehouse: my_warehouse
    prod:
      type: snowflake
      account: 'not needed'
      user: 'not needed'
      role: my_role
      database: my_database
      schema: prod
      warehouse: my_warehouse
```

**Critical**: The target schema must exist before creating the dbt project object. Snowflake dbt does NOT auto-create schemas (unlike standard dbt Core).

### dbt_project.yml

Standard dbt configuration. The `profile` field must reference a profile name defined in `profiles.yml`:

```yaml
name: 'my_project'
version: '1.0.0'
profile: 'my_project'

model-paths: ["models"]
seed-paths: ["seeds"]
test-paths: ["tests"]
macro-paths: ["macros"]
snapshot-paths: ["snapshots"]
```

## Snowflake CLI Commands (`snow dbt`)

**Requires Snowflake CLI version 3.13.0 or later.**

### Install Snowflake CLI

```bash
# macOS
brew install snowflake-cli

# pip
pip install snowflake-cli
```

### Deploy a dbt Project

Uploads local files to a temporary stage and creates/updates a dbt project object.

```bash
# Basic deploy
snow dbt deploy my_project

# Deploy with options
snow dbt deploy my_project \
  --source /path/to/dbt/directory \
  --profiles-dir ~/.dbt/ \
  --default-target prod \
  --external-access-integration my_ext_integration \
  --force

# Force recreate existing project
snow dbt deploy my_project --force
```

**Flags:**
| Flag | Description |
|------|-------------|
| `--source` | Path to dbt project directory |
| `--profiles-dir` | Path to directory containing `profiles.yml` |
| `--default-target` | Default target environment (e.g., `dev`, `prod`) |
| `--external-access-integration` | External access integration name (repeatable) |
| `--force` | Force create or recreate the project |

### Execute dbt Commands

Run any standard dbt command against a deployed project object.

```bash
# Run all models
snow dbt execute my_project run

# Run specific models
snow dbt execute my_project run --select customer_metrics

# Run with downstream dependencies
snow dbt execute my_project run --select customer_metrics+

# Test models
snow dbt execute my_project test

# Install dependencies
snow dbt execute my_project deps

# Compile (generate SQL without running)
snow dbt execute my_project compile

# Run asynchronously (non-blocking)
snow dbt execute --run-async my_project run --select @source:snowplow,tag:nightly

# Seed data
snow dbt execute my_project seed

# Snapshot
snow dbt execute my_project snapshot

# Target override
snow dbt execute my_project run --target prod
```

**Supported dbt commands:** `build`, `compile`, `deps`, `list`, `parse`, `retry`, `run`, `run-operation`, `seed`, `show`, `snapshot`, `test`

### List dbt Projects

```bash
# List all projects
snow dbt list

# Filter by pattern
snow dbt list --like JAFFLE%

# Filter by database
snow dbt list --like JAFFLE% --in database product
```

### Describe a Project

```bash
snow dbt describe my_project
```

### Drop a Project

```bash
snow dbt drop my_project
```

## SQL Commands Reference

### CREATE DBT PROJECT

```sql
CREATE [ OR REPLACE ] DBT PROJECT [ IF NOT EXISTS ] <name>
  [ FROM '<source_location>' ]
  [ COMMENT = '<string_literal>' ]
  [ DEFAULT_TARGET = <default_target> ]
  [ EXTERNAL_ACCESS_INTEGRATIONS = ( <integration_name> [ , ... ] ) ]
```

**Source location types:**
- Git repository stage: `'@db.schema.git_repo_stage/branches/main/path'`
- Existing dbt project: `'snow://dbt/db.schema.project/versions/last'`
- Internal named stage: `'@db.schema.my_stage/path'`
- Workspace: `'snow://workspace/user$.public."workspace_name"/versions/live/path'`

**Examples:**

```sql
-- From Git repository
CREATE DBT PROJECT sales_db.dbt_schema.sales_model
  FROM '@sales_db.integrations.git_stage/branches/main'
  DEFAULT_TARGET = 'prod'
  EXTERNAL_ACCESS_INTEGRATIONS = ('my_ext_integration')
  COMMENT = 'Sales data models';

-- From subdirectory in Git
CREATE DBT PROJECT sales_db.dbt_schema.regional_model
  FROM '@sales_db.integrations.git_stage/branches/main/regional_project'
  DEFAULT_TARGET = 'prod'
  EXTERNAL_ACCESS_INTEGRATIONS = ('my_ext_integration');

-- From existing dbt project version
CREATE DBT PROJECT sales_db.dbt_schema.sales_model_v2
  FROM 'snow://dbt/sales_db.dbt_schema.sales_model/versions/version$2'
  DEFAULT_TARGET = 'prod';

-- From workspace
CREATE DBT PROJECT sales_db.dbt_schema.workspace_model
  FROM 'snow://workspace/user$.public."My Workspace"/versions/live/project';

-- Empty project (add files later)
CREATE DBT PROJECT my_db.my_schema.empty_project;
```

**Access control**: Requires CREATE DBT PROJECT privilege on the schema.

### ALTER DBT PROJECT

```sql
-- Rename
ALTER DBT PROJECT [ IF EXISTS ] <name> RENAME TO <new_name>

-- Add a new version (auto-increments version$N)
ALTER DBT PROJECT <name> ADD VERSION [ <version_alias> ]
  FROM '<source_location>'

-- Set properties
ALTER DBT PROJECT [ IF EXISTS ] <name> SET
  [ DEFAULT_TARGET = <default_target> ]
  [ EXTERNAL_ACCESS_INTEGRATIONS = ( <integration_name> [ , ... ] ) ]
  [ COMMENT = '<string_literal>' ]

-- Unset properties
ALTER DBT PROJECT [ IF EXISTS ] <name> UNSET
  [ DEFAULT_TARGET ]
  [ EXTERNAL_ACCESS_INTEGRATIONS ]
  [ COMMENT ]
```

**Examples:**

```sql
-- Update from latest Git (fetch first, then add version)
ALTER GIT REPOSITORY sales_db.integrations.git_stage FETCH;
ALTER DBT PROJECT sales_db.dbt_schema.sales_model
  ADD VERSION
  FROM '@sales_db.integrations.git_stage/branches/main/sales_project';

-- Add version with alias
ALTER DBT PROJECT sales_db.dbt_schema.sales_model
  ADD VERSION release_v2
  FROM '@sales_db.integrations.git_stage/branches/main';

-- Change default target
ALTER DBT PROJECT sales_db.dbt_schema.sales_model SET
  DEFAULT_TARGET = 'prod',
  EXTERNAL_ACCESS_INTEGRATIONS = ('my_ext_integration');
```

**Access control**: Requires OWNERSHIP privilege.

### EXECUTE DBT PROJECT

```sql
-- Execute a dbt project object
EXECUTE DBT PROJECT [ IF EXISTS ] <name>
  [ ARGS = '<dbt_command> [ --<option> <value> ... ]' ]

-- Execute from workspace
EXECUTE DBT PROJECT [ IF EXISTS ] [ FROM WORKSPACE <workspace_name> ]
  [ ARGS = '<dbt_command> [ --<option> <value> ... ]' ]
  [ PROJECT_ROOT = '<subdirectory_path>' ]
```

**Output columns:**
| Column | Description |
|--------|-------------|
| `Success` | TRUE/FALSE |
| `EXCEPTION` | Error message or "None" |
| `STDOUT` | dbt output |
| `OUTPUT_ARCHIVE_URL` | URL to logs and `/target` artifacts |

**Examples:**

```sql
-- Run all models with dev target
EXECUTE DBT PROJECT my_db.my_schema.my_project
  ARGS = 'run --target dev';

-- Run specific models
EXECUTE DBT PROJECT my_db.my_schema.my_project
  ARGS = 'run --select customer_metrics orders --target dev';

-- Run downstream dependencies
EXECUTE DBT PROJECT my_db.my_schema.my_project
  ARGS = 'run --select customer_metrics+ --target dev';

-- Test models
EXECUTE DBT PROJECT my_db.my_schema.my_project
  ARGS = 'test --target prod';

-- Install dependencies (requires external access integration)
EXECUTE DBT PROJECT my_db.my_schema.my_project
  ARGS = 'deps --target dev'
  EXTERNAL_ACCESS_INTEGRATIONS = (dbt_ext_access);

-- Compile only (no execution)
EXECUTE DBT PROJECT my_db.my_schema.my_project
  ARGS = 'compile --target dev';

-- Execute from workspace
EXECUTE DBT PROJECT FROM WORKSPACE user$.public."my_workspace"
  PROJECT_ROOT = 'my_dbt_project'
  ARGS = 'run --target dev';
```

**Access control**: Requires USAGE privilege on the dbt project object. Role in `profiles.yml` must have privileges on warehouse, database, schema, and referenced objects.

### DESCRIBE DBT PROJECT

```sql
DESCRIBE DBT PROJECT <name>;
```

### DROP DBT PROJECT

```sql
DROP DBT PROJECT [ IF EXISTS ] <name>;
```

**Access control**: Requires OWNERSHIP privilege.

### SHOW DBT PROJECTS

```sql
SHOW DBT PROJECTS [ LIKE '<pattern>' ]
  [ IN { ACCOUNT | DATABASE | DATABASE <db> | SCHEMA | SCHEMA <schema> } ]
  [ STARTS WITH '<prefix>' ]
  [ LIMIT <rows> [ FROM '<name>' ] ];
```

**Output columns include:** `name`, `database_name`, `schema_name`, `created_on`, `updated_on`, `owner`, `comment`, `dbt_snowflake_version`, `default_target`, `default_version`, `default_version_name`, `default_version_alias`, `external_access_integrations`

```sql
-- Show all projects in a database
SHOW DBT PROJECTS IN DATABASE my_db;

-- Filter by pattern
SHOW DBT PROJECTS LIKE 'sales%';
```

### SHOW VERSIONS IN DBT PROJECT

```sql
SHOW VERSIONS IN DBT PROJECT <name> [ LIMIT <rows> ];
```

**Output columns:** `created_on`, `name` (e.g., VERSION$1), `alias`, `location_uri`, `is_default`, `is_live`, `is_first`, `is_last`, `source_location_uri`, `git_commit_hash`

## Git Integration Setup

### Prerequisites for Git-sourced dbt Projects

```sql
-- 1. Create database and schemas
CREATE DATABASE my_dbt_db;
CREATE SCHEMA my_dbt_db.integrations;
CREATE SCHEMA my_dbt_db.dev;
CREATE SCHEMA my_dbt_db.prod;

-- 2. Create secret for private repos (skip for public)
CREATE OR REPLACE SECRET my_dbt_db.integrations.git_secret
  TYPE = password
  USERNAME = 'github-username'
  PASSWORD = 'ghp_your_personal_access_token';

-- 3. Create API integration
CREATE OR REPLACE API INTEGRATION git_api_integration
  API_PROVIDER = git_https_api
  API_ALLOWED_PREFIXES = ('https://github.com/my-org')
  ALLOWED_AUTHENTICATION_SECRETS = (my_dbt_db.integrations.git_secret)
  ENABLED = TRUE;

-- 4. Create external access integration (for dbt deps)
CREATE OR REPLACE NETWORK RULE dbt_network_rule
  MODE = EGRESS
  TYPE = HOST_PORT
  VALUE_LIST = ('hub.getdbt.com', 'codeload.github.com');

CREATE OR REPLACE EXTERNAL ACCESS INTEGRATION dbt_ext_access
  ALLOWED_NETWORK_RULES = (dbt_network_rule)
  ENABLED = TRUE;
```

### Enable Logging and Tracing

```sql
ALTER SCHEMA my_dbt_db.dev SET LOG_LEVEL = 'INFO';
ALTER SCHEMA my_dbt_db.dev SET TRACE_LEVEL = 'ALWAYS';
ALTER SCHEMA my_dbt_db.dev SET METRIC_LEVEL = 'ALL';
```

## Scheduling dbt Runs with Tasks

```sql
-- Scheduled run every hour at minute 01
CREATE OR REPLACE TASK my_db.my_schema.hourly_dbt_run
  WAREHOUSE = my_warehouse
  SCHEDULE = 'USING CRON 1 * * * * America/Los_Angeles'
AS
  EXECUTE DBT PROJECT my_db.my_schema.my_dbt_project
    ARGS = 'run --target prod';

-- Chained task: run then test
CREATE OR ALTER TASK my_db.my_schema.run_dbt
  WAREHOUSE = my_warehouse
  SCHEDULE = '6 hours'
AS
  EXECUTE DBT PROJECT my_db.my_schema.my_project
    ARGS = 'run --target prod';

CREATE OR ALTER TASK my_db.my_schema.test_dbt
  WAREHOUSE = my_warehouse
  AFTER my_db.my_schema.run_dbt
AS
  EXECUTE DBT PROJECT my_db.my_schema.my_project
    ARGS = 'test --target prod';

-- Run specific models on schedule
CREATE OR REPLACE TASK my_db.my_schema.run_metrics
  WAREHOUSE = my_warehouse
  SCHEDULE = 'USING CRON 1 * * * * America/Los_Angeles'
AS
  EXECUTE DBT PROJECT my_db.my_schema.my_project
    ARGS = 'run --select customer_loyalty_metrics --target prod';

-- Resume tasks (tasks are created suspended by default)
ALTER TASK my_db.my_schema.hourly_dbt_run RESUME;
```

## CI/CD Integration with GitHub Actions

When building CI/CD workflows, you only need your Git server and Snowflake CLI. A Git repository object in Snowflake is NOT required.

```yaml
name: dbt Deploy and Test

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Snowflake CLI
        run: pip install snowflake-cli

      - name: Configure Snowflake connection
        run: |
          mkdir -p ~/.snowflake
          cat > ~/.snowflake/config.toml <<EOF
          [connections.default]
          account = "${{ secrets.SNOWFLAKE_ACCOUNT }}"
          user = "${{ secrets.SNOWFLAKE_USER }}"
          password = "${{ secrets.SNOWFLAKE_PASSWORD }}"
          role = "MY_ROLE"
          warehouse = "MY_WAREHOUSE"
          database = "MY_DATABASE"
          schema = "MY_SCHEMA"
          EOF

      - name: Deploy dbt project
        run: snow dbt deploy my_project --source ./dbt --force

      - name: Run dbt models
        run: snow dbt execute my_project run --target prod

      - name: Test dbt models
        run: snow dbt execute my_project test --target prod
```

## Common Workflows

### Local Development Cycle

```bash
# 1. Make changes to dbt models locally
# 2. Deploy to Snowflake (dev)
snow dbt deploy my_project --source . --default-target dev

# 3. Install dependencies
snow dbt execute my_project deps

# 4. Compile to check SQL
snow dbt execute my_project compile

# 5. Run models
snow dbt execute my_project run --target dev

# 6. Run tests
snow dbt execute my_project test --target dev

# 7. When ready, deploy to prod
snow dbt execute my_project run --target prod
snow dbt execute my_project test --target prod
```

### Update Deployed Project from Git

```sql
-- Fetch latest from Git
ALTER GIT REPOSITORY my_db.integrations.git_stage FETCH;

-- Add new version to project
ALTER DBT PROJECT my_db.my_schema.my_project
  ADD VERSION
  FROM '@my_db.integrations.git_stage/branches/main/dbt_project';

-- Execute the updated project
EXECUTE DBT PROJECT my_db.my_schema.my_project
  ARGS = 'run --target prod';
```

### Verify Deployed Objects

```sql
-- Check project exists
SHOW DBT PROJECTS LIKE 'my_project%';

-- Check versions
SHOW VERSIONS IN DBT PROJECT my_db.my_schema.my_project;

-- Describe project details
DESCRIBE DBT PROJECT my_db.my_schema.my_project;

-- Verify materialized tables and views
SHOW TABLES IN SCHEMA my_db.prod;
SHOW VIEWS IN SCHEMA my_db.prod;
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "Schema does not exist" on execute | Create target schema before deploying: `CREATE SCHEMA IF NOT EXISTS db.schema` |
| Password in profiles.yml | Remove passwords — Snowflake native dbt does not support them |
| `dbt deps` fails | Ensure EXTERNAL_ACCESS_INTEGRATIONS is set with network rules allowing `hub.getdbt.com` and `codeload.github.com` |
| Insufficient privileges | Check that the role in `profiles.yml` has USAGE on warehouse, database, and schema |
| Version not updating | Run `ALTER GIT REPOSITORY ... FETCH` before `ALTER DBT PROJECT ... ADD VERSION` |
| CLI version error | Upgrade to Snowflake CLI >= 3.13.0: `pip install --upgrade snowflake-cli` |
